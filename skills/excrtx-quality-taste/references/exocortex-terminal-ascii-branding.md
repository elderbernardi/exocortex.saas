# Exocortex terminal-native branding cues

Use this reference when creating ASCII logos, terminal headers, splash screens, or monochrome identity assets for Exocórtex.

## Brand intent
- Exocórtex should feel terminal-native, precise, and cognitive — not playful, cartoonish, or cyberpunk-noisy.
- The mark should converse with Hermes interfaces on web and desktop without depending on UI chrome.
- The identity should remain legible in both dark and light themes.

## Durable visual constraints
- Design monochrome-first. ANSI color is optional garnish, never structural.
- Prioritize silhouette, spacing, rhythm, and negative space over fill density.
- The mark must survive plain ASCII rendering; box-drawing characters are an optional enhanced variant, not the only variant.
- Avoid heavy blocks that disappear on light backgrounds or look muddy on dark ones.
- Prefer balanced horizontal compositions that fit terminal headers and app splash areas.

## Exocortex-specific cues
- Favor the wordmark idea `exo[cortex]` or close descendants when the logo needs to expose the product name directly.
- Keep the relation to Hermes subtle: companion layer, cognitive shell, orchestration surface.
- When a signature line is needed, use `powered by Hermes Agent` as a secondary line, never as the main mark.
- Palette guidance for colored variants: cyan / petróleo accents are acceptable, but the base composition must work with zero color.

## Layout targets
Prepare or evaluate concepts against at least these widths:
- compact: ~24–32 columns
- standard: ~40–60 columns
- wide hero: ~72+ columns

The compact version should still read as intentional rather than collapsed residue from the wide version.

## Review criteria
Reject candidates that:
- depend on ANSI color to make sense
- look like generic hacker ASCII with random slashes and noise
- become illegible on white backgrounds
- use dense ornament where structure should carry the identity
- feel more like a banner than a reusable mark

Prefer candidates that:
- read cleanly in JetBrains Mono, Fira Code, and default system monospace fonts
- preserve a memorable center of gravity even when copied into plain text
- can anchor both CLI output and product chrome without visual conflict
