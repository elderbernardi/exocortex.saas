# ASCII Terminal Branding — Exocortex x Hermes

Captured context:
- The user rejected ASCII logo versions that looked like diagrams, flowcharts, or terminal fan art.
- For the "terminal-native" path, the approved direction was restraint: dry wordmark, clean signature, and composition that looks like mature software.

Operational lessons:
1. In ASCII branding, too much detail usually makes it worse. Professionalism comes from typographic discipline, whitespace, and hierarchy.
2. If the brand dialogues with Hermes Agent, Hermes enters as a runtime layer: secondary signature, not dominant title.
3. The text "Hermes Agent" must remain clean. Don't deform the wordmark into ornamental ASCII when it appears in lockups like "powered by Hermes Agent."
4. Exocortex needs to separate its palette from the Hermes identity. Avoid amber/gold as the main Exocortex color.
5. When the brand's philosophy is already encoded in the name itself, explore that before designing symbols. Useful example: `exo[cortex]` as an extension wrapping the core.

Error signals:
- The result looks like a technical diagram instead of a brand.
- The symbol over-explains and kills memorability.
- The ASCII relies on heavy visual mass (figlet-style) to seem important.
- The composition competes with Hermes instead of framing it as a base.

Preferred direction:
- Main wordmark: minimalist, terminal-native, legible on dark and light.
- Secondary lockup: `powered by Hermes Agent` in a subordinate line.
- Micro mark: a short, usable form for headers, prompts, and badges.

Valid direction example:
- `exo[cortex]`
- `exo[cortex] :: powered by Hermes Agent`
- Micro mark: `[cx]`

Suggested palette to differentiate from Hermes:
- Dark background: `#0B1116`
- Surface: `#101A22`
- Accent: `#62E6FF`
- Accent deep: `#1AA7C8`
- Light text: `#EAF7FB`
- Muted text: `#89A3AE`

Rule of thumb:
If it feels like you're "drawing an idea," step back. In brand ASCII, disciplined typography almost always beats overly clever illustration.
