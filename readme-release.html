🏛️ System Philosophy

The Exocortex operates on a fundamental premise: LLMs have vast knowledge of the past but lack intent and are blind to your immediate present. The Exocortex acts as the structural bridge — translating your intent and context to govern and focus the processing power of the AI.

graph TD A[Executive / Intent] -->|Governs| B(Exocórtex.IA / Method) B -->|Operates| C(Hermes Agent / Execution) C -->|Orchestrates| D[62 Skills, MCPs, Acervo, APIs]

1. The Three Concentric Layers

To eliminate semantic drift and optimize contextual efficiency, all information and operations are organized into three concentric depth levels:

  • 🏛️ Macroverso (Who Speaks): The executive's personal "Constitution." Defines core identity, non-negotiable values, communication style, tone, and personal boundaries. Generated during onboarding; rarely changes; silently governs all interactions.
  • 🌍 Microversos (Semantic Domains): Live, self-contained semantic and operational entities — specific clients, projects, disciplines, or areas of responsibility. Each preserves its own context, rules, memory, and sharing constraints (deny: [ALL], allow: [gabinete], where allow takes precedence over deny).
  • 🎯 Tarefa (The Operational Room): The concrete room where execution happens. Short-lived. Anchored to a primary Microverso and may pull secondary Microversos for support. Rule EX-06: A Microverso is never a room; the Tarefa is the room.

2. The Three Operational Vectors

The Exocortex dynamically adjusts its cognitive posture by classifying every executive interaction into one of three operational vectors:

Vector Cognitive Posture Focus Behavior
🧠 Evolução (THINK) Socratic Guide Idea refinement & understanding Challenges assumptions. Asks 2-3 deep analytical questions before concluding. Promotes learnings to the Acervo. Never gives lazy, ready-made answers.
⚡ Execução (DO) Specialist Agent Production of premium deliverables Fast, precise, highly technical. Builds documents, drafts code, coordinates execution with quality validation.
🧹 Manutenção (CLEAN) Ecological Housekeeper Ecosystem health & integrity Background verification: quality audits, index updates, stale log archival, path/manifest validation. Autonomous síndico cron job per ADR-018.

3. Core Governance Protocols

Draft-First Protocol

Irreversible actions external to the local execution environment — sending emails, scheduling calendar events, pushing to Git, publishing posts, deploying code — must never be automated directly. The Exocortex prepares the payload, presents it as a 📋 DRAFT, and waits for explicit executive approval before executing.

Accuracy Verification (EX-49)

The Exocortex is forbidden from claiming a system action succeeded without executing an empirical verification command and printing the raw output as physical proof. "Feito" requires evidence.

Anti-Slop Quality Gates (EX-18/EX-19)

Every piece of prose is scored on Directness, Rhythm, Trust, Authenticity, and Density. Minimum: 35/50. Visual output is filtered for generic templates, meta-labels, and empty grids. AI patterns are rejected — the executive's voice is preserved.

⚙️ Installation

Prerequisites

  • OS: Linux (Debian, Ubuntu, Arch) or macOS.
  • System Tools: git, curl, rsync, python3 (≥3.11), pip, venv.
  • Containers: docker & docker compose (required only if Hindsight or Firecrawl are enabled).
🤖 AI Agents: If you are an AI agent (Claude, Gemini, GPT, etc.) installing or diagnosing this system via terminal, use INSTALL.md instead — a structured runbook with pre-conditions, executable commands, and verification steps designed for machine consumption.

1. Bootstrap Installer

Downloads the installer, checks OS dependencies, installs the Hermes Agent CLI (if absent), and clones the Exocortex repository:

# Standard interactive installation
curl -fsSL https://raw.githubusercontent.com/elderbernardi/exocortex.saas/main/install.sh | bash

# With Telegram Gateway auto-provisioned
TELEGRAM_BOT_TOKEN="your_token" curl -fsSL https://raw.githubusercontent.com/elderbernardi/exocortex.saas/main/install.sh | bash

# Pin a specific version
VERSION=v1.1.1 curl -fsSL https://raw.githubusercontent.com/elderbernardi/exocortex.saas/main/install.sh | bash

2. Setup Orchestrator

The installer invokes setup.sh automatically. Customize home paths manually:

HERMES_HOME=~/.hermes EXOCORTEX_HOME=~/exocortex bash setup.sh

Enable optional services via environment variables:

# Hermes WebUI cockpit (MIT, port 8787)
EXOCORTEX_ENABLE_HERMES_WEBUI=1 bash setup.sh

# Hindsight operational memory (Docker)
EXOCORTEX_ENABLE_HINDSIGHT=1 bash setup.sh

# Firecrawl self-host tiered scraping (Docker)
EXOCORTEX_ENABLE_FIRECRAWL=1 bash setup.sh

# Context7 tech-docs MCP
EXOCORTEX_ENABLE_CONTEXT7=1 bash setup.sh

What setup.sh executes (15 steps):

  1. Validates Hermes version compatibility.
  2. Provisions optional Docker services (Hindsight, Firecrawl).
  3. Initializes directory trees — workspace, logs, task boards, 4-layer Acervo.
  4. Copies and installs all 62 excrtx skills, bundles, and execution profiles.
  5. Hardening: patches Google Drive search pagination, removes legacy skills, replaces Composio with direct API clients.
  6. Google OAuth setup, DocBrain engine clone & build, NotebookLM CLI, Browser automation, Hermes WebUI (optional), Context7 MCP (optional).
  7. LLM key verification — real ping per role, config.yaml write, model-ID format validation.
  8. Creates idempotent maintenance crons (síndico per ADR-018).
  9. Final skill mapping verification and calibration prompt.

🛠️ Post-Installation

1. Onboarding & Calibration

On first launch, the Exocortex detects an empty SOUL.md and triggers the welcome flow:

hermes                          # launch interactive session
vamos começar o onboarding     # initiate 6-block interview

The interview covers Identity, Communication, Domains, Business Context, Operational Preferences, and Integrations — writing your personality profile to the Macroverso.

2. Behavioral Calibration

Ensure the LLM respects the 12 core behavioral features (Vector classification, Draft-first, verification proofs, etc.):

bash scripts/calibrate-hermes.sh

3. Google Workspace OAuth

Desktop OAuth flow — configure GCP Console, download client secret, then:

cp /path/to/secret.json ~/.hermes/google_client_secret.json
python3 ~/.hermes/skills/productivity/google-workspace/scripts/setup.py --client-secret ~/.hermes/google_client_secret.json
python3 ~/.hermes/skills/productivity/google-workspace/scripts/setup.py --auth-url
# → open URL in browser, authorize, copy the code= parameter
python3 ~/.hermes/skills/productivity/google-workspace/scripts/setup.py --auth-code "PASTE_CODE"
python3 ~/.hermes/skills/productivity/google-workspace/scripts/setup.py --check
# → AUTHENTICATED

4. NotebookLM CLI

uv tool install notebooklm-mcp-cli --force
nlm login
hermes mcp add notebooklm --command notebooklm-mcp

5. DocBrain Parser Engine

cd ~/exocortex/tools/docbrain
npm install && npm run build

6. Browser Automation

cd ~/.hermes/skills/excrtx/excrtx-integrate-browser
bash scripts/browser-use.sh open https://example.com
# → auto-provisions Playwright + Chromium on first run
