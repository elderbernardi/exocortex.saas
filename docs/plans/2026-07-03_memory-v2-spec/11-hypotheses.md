# 11 — Open Hypotheses & Research Questions

> What v2 decides *provisionally* and how to find out if it decided wrong. Each: position taken, why it matters, experiment, decision metric, cost of being wrong.

**H1 — Markdown-first vs database-first.** *Position: Markdown canonical (P1).* Risk: at 10k+ objects, file-scan workflows degrade. **Experiment:** synthetic acervo at 10k/50k objects; measure catalog rebuild, reconcile, retrieval latency. **Metric:** p95 targets in 10 §2 hold at 10k. **If wrong:** catalog becomes read-path primary (files stay canonical) — the architecture already permits this; that's why the catalog exists. *Low risk.*

**H2 — Agentic search vs semantic index at personal scale.** *Position: agentic+lexical primary, Hindsight supplement (07 §1).* Disputed in the field (Claude Code vs Milvus positions). **Experiment:** run golden set 3 ways (catalog+grep only / Hindsight only / hybrid). **Metric:** recall & token cost per category; if hybrid ≤ +5% recall over catalog+grep, demote Hindsight to optional. **If wrong either way:** routing table weights change — no structural cost. *This experiment is cheap and should run in Phase 3.*

**H3 — Claim-level vs file-level memory.** *Position: file is the atom; claim anchors optional (05 §1).* **Experiment:** for one contradiction-heavy microverso (pricing/comercial), maintain claim anchors for 60 days; count conflicts caught at claim level that file level would have missed, vs annotation overhead. **Metric:** >20% extra catches → expand; <5% → drop anchors entirely. **Risk if wrong:** over-atomization = junk + write friction (the A-MEM/Zettelkasten failure mode for humans); under-atomization = mushy supersedence.

**H4 — Graph value below corpus scale.** *Position: thin graph (5 edge types + wikilinks), no graph DB (04 §2).* **Experiment:** add multi-hop questions to golden set ("qual cliente foi afetado pela decisão que substituiu X?"); measure failure rate of thin-graph traversal. **Metric:** multi-hop recall < 0.7 → build PPR-style traversal over catalog links (still SQLite, HippoRAG-2-lite). **Risk:** premature graph = the GraphRAG cost trap; late graph = a few missed syntheses. Late is cheaper.

**H5 — Microverso as hard boundary vs soft tag.** *Position: hard namespace with declared bridges (06).* **Experiment:** contamination + cross-scope-synthesis metrics together — the tension is real: isolation hurts synthesis. **Metric:** contamination must stay 0 while allowed-path synthesis recall ≥ 0.8; if synthesis fails, loosen via groups/expose, never via removing the scope guard. **Risk if wrong:** too hard = the exocortex can't think across the executive's life (its whole point); too soft = the one unforgivable failure (client leakage).

**H6 — Consolidation rewrite vs annotate.** *Position: annotate + supersede, git-auditable (04 §4); rewriting only for context heads/entity profiles.* OpenAI's Dreaming shows rewriting lifts recall; its critics show the audit cost. **Experiment:** A/B on entity pages — rewritten-profile vs append-only + generated summary; measure answer quality + executive trust in diffs. **Metric:** if rewrite wins on quality and diffs stay reviewable, expand rewrite class; any confabulation incident → freeze.

**H7 — Governed auto-commit vs approval-first.** *Position: risk-tiered auto-commit (08 §2).* **Experiment:** 30 days of journal audit: % of auto-committed objects later corrected/reverted by the executive. **Metric:** >10% corrected → tighten gates; <2% → widen (e.g. auto-approve global knowledge too). **Risk:** too tight = system stops learning (rubber-stamp fatigue); too loose = poisoning/drift.

**H8 — Eager vs lazy summarization.** *Position: lazy by default; precompute only descriptions, index lines, entity summaries (P8).* **Experiment:** token cost + latency of summarize-on-read for the briefing (the heaviest reader) vs nightly pre-digest. **Metric:** briefing p95 > 60s or cost > threshold → precompute briefing inputs nightly.

**H9 — When does a conversation become an episode?** *Position: significance gate = decision ∨ commitment ∨ artifact ∨ executive flag (07 §3).* **Experiment:** 30 days; count "onde paramos"-type questions that missed because no episode existed. **Metric:** >1 miss/week → lower the gate (e.g. any session > N turns in an active microverso).

**H10 — When does a reflection become a rule?** *Position: never automatically; executive sign-off (08 §5).* **Experiment:** track reflections repeatedly retrieved in Execução tasks (≥3 retrievals) — propose promotion in digest. **Metric:** acceptance rate of proposals; if executive accepts >80%, keep proposing at 3; if he's annoyed, raise threshold.

**H11 — LLM-maintained vs human-maintained ontology.** *Position: vocabulary (types, epistemic, facets) is human-governed and versioned; instances are agent-maintained.* **Experiment:** quarterly — count objects that fit no type well (journal `type: knowledge` + tag `misc-fit`). **Metric:** >5% misfits → vocabulary review ADR. **Risk:** LLM-evolved vocabulary drifts silently; frozen vocabulary makes the schema lie.

**H12 — What is "useful memory"?** *Position (operational definition): memory is useful iff it is retrieved into a task that the executive doesn't correct.* **Experiment:** log retrieval→usage→correction chains (journal has all three). **Metric:** per-object usefulness = retrievals without correction; syndic gains a "never retrieved in 180d and class volátil" demotion signal — replacing pure time-decay with **use-decay**. This is the long-term learning loop of the whole system.
