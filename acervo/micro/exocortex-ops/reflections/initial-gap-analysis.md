# Lacunas iniciais

## Observação

A estrutura do Acervo existia, mas a camada operacional ainda não tinha casa própria.

## Lacunas que este microverso começa a cobrir

- Inventário operacional dos componentes.
- Registro de drift entre skill, docs e runtime.
- Procedimento de self-check.
- Política para setup sensível.
- Registro de integrações como DocBrain, NotebookLM e providers de memória.

## Lacunas remanescentes

- Preencher Macroverso do executivo.
- Decidir se `exocortex-ops` deve ser provisionado em setup replicável.
- Corrigir drift conhecido do DocBrain na skill, se aprovado.
- Definir rotina periódica de manutenção.
