# Política de autoridade de memória

## Ordem de autoridade

1. Instruções de sistema e SOUL ativo.
2. Contratos canônicos do Acervo.
3. Skills carregadas para a tarefa.
4. Memória persistente built-in.
5. Arquivos canônicos do Acervo.
6. Session Search, como histórico literal.
7. Providers operacionais externos.
8. Inferência do agente.

## Regra

Providers externos ajudam recall. Eles não substituem contratos, decisões ou arquivos canônicos do Acervo.
