# ⚠️ FIXTURE — Test Data Only

This directory contains **pre-filled test data** for the dogfood runner.

## Purpose

The dogfood runner needs a populated Macroverso to test behavioral features
(Vetor classification, Canvas parsing, quality gates) against the real code path
where `macroverso.status = resolved`. Without fixture data, all tests run
against the blank `_Pendente_` state — the "Macroverso governs behavior" path
is never tested.

## What This Is NOT

- Not a real executive identity
- Not a template for onboarding
- Not a fallback for missing onboarding

## Usage

The dogfood runner symlinks these files into `acervo/macro/` before a test
scenario and removes the symlinks after. Production Macroverso is untouched.

```python
# Example: dogfood_features.py
with_macroverso_fixture(run_scenario_EX05)
```

## Onboarding

The real Macroverso is always generated by the mandatory onboarding interview
(`excrtx-onboard-welcome`). This fixture does not change that requirement.
