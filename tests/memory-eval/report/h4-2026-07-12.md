# H4 — Graph value below corpus scale (run 2026-07-12)

**Hypothesis (11-hypotheses.md H4):** a thin graph (5 edge types + wikilinks, no
graph DB) is enough at this corpus scale. **Metric:** multi-hop recall < 0.70 ⇒
build a PPR-style traversal over catalog links (HippoRAG-2-lite); otherwise the
thin graph suffices and Phase 5 stays cut.

Retrieval-only scorer over the synthetic fixture (`tests/memory-eval/fixture/`,
25 golden questions, top-k=5). Deterministic, offline — the `production`
strategy is the Phase 3 shipping retrieval (`acervo_retrieve.retrieve`) called
in-process; with the Hindsight container down it degrades to the catalog+grep
ladder (the fallback path), so these numbers are the floor, not a ceiling.

## Verdict

**H4 does NOT trigger.** Multi-hop recall is 100% on the shipping retrieval and
83.3% even on the lexical-only fallback — both well above the 0.70 threshold.
**Phase 5 (PPR graph traversal) is not justified** on the current evidence; the
thin graph + scope-bridge routing already resolves every multi-hop synthesis
question in the fixture. This matches the roadmap's cut order ("cut P5 first;
late graph is cheaper").

Bonus (H5 corroboration): contamination stayed 0 while allowed-path synthesis
recall hit 100% — the hard scope boundary is not blocking cross-scope thinking
on this fixture.
